# Guía de contribución

## Ramas
- `main` → producción (solo merge desde release)
- `develop` → integración principal

## Flujo
1. Crea tu rama desde `develop`: `feature/nombre`
2. Haz commits descriptivos
3. Abre un Pull Request hacia `develop`
4. Espera 1 aprobación antes de hacer merge
5. Elimina tu rama después del merge

## Nombrado de ramas
- feature/descripcion-corta
- fix/descripcion-corta
- hotfix/descripcion-corta
